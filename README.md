Многопоточный анализ текстовых файлов

**Описание**

Приложение принимает список файлов через аргументы командной строки или, если аргументы отсутствуют, автоматически анализирует все `.txt` файлы в текущей директории. Для каждого файла выполняется:

- асинхронное чтение содержимого (используется `std::async`);
- последующий анализ содержимого в пуле потоков (`ThreadPool`), где считается количество слов и символов;
- запись результата в общий вектор с защитой `std::mutex`.

В случае ошибки чтения файла (файл не найден, нет доступа и т.д.) приложение не падает — в результатах отмечается ошибка для соответствующего файла.

**Реализованные требования**

1) Структура данных для хранения результатов анализа
- `FileAnalysis` — содержит имя файла, число слов, число символов, флаг успеха и поле с описанием ошибки.

2) Многопоточная обработка
- Реализован легковесный `ThreadPool` в `src/thread_pool.h`.
- Чтение файлов выполняется асинхронно (`std::async`), а анализ — задачами в пуле потоков.
- Для синхронизации доступа к вектору результатов используется `std::mutex`.

3) Асинхронная обработка и обработка ошибок
- Чтение файла вынесено в функцию `readFile()`; ошибки чтения бросают `std::runtime_error`, которые затем корректно обрабатываются.
- Если чтение завершается с ошибкой, в `FileAnalysis` записывается сообщение об ошибке.

4) Вывод результатов
- Приложение выводит список обработанных файлов с числом слов и символов по каждому файлу и итоговую сумму по всем успешно обработанным файлам.

**Структуры и ключевые функции**

- `struct FileAnalysis` — хранит `filename`, `words`, `characters`, `ok`, `error`.
- `analyzer::collectInputFiles(int argc, char** argv)` — собирает список файлов из аргументов или `.txt` в текущей директории.
- `analyzer::analyzeFiles(const std::vector<std::string>& files)` — отвечает за асинхронное чтение, многопоточный анализ и сбор результатов.
- `analyzer::printAnalysisResults(const std::vector<FileAnalysis>& results)` — форматированный вывод итогов.
- `ThreadPool` — простая реализация пула потоков (submit -> `std::future`).

Реализация подсчёта слов использует `std::istringstream` и оператор `>>`, поэтому разделителями считаются пробельные символы; если нужен более строгий или независимый от локали подсчёт слов (например, учитывать дефисы или юникод), см. раздел "Возможные улучшения".

**Сборка**

Требования:
- Компилятор, совместимый с C++17 (GCC, Clang)
- CMake 3.10+

Команды для сборки (в корне проекта):

```bash
mkdir -p build
cmake -S . -B build
cmake --build build -- -j
```

или коротко:

```bash
cmake -S . -B build && cmake --build build -- -j
```

**Запуск**

1) Передать список файлов аргументами:

```bash
./build/file_analyzer test_files/file1.txt test_files/file2.txt
```

2) Запустить без аргументов — будут обработаны все `.txt` в текущей директории:

```bash
./build/file_analyzer
```

Программа печатает результаты в консоль.

**Примеры вывода**

Пример (запуск на `test_files/*`):

```
Результаты анализа:
1. test_files/file1.txt: 21 слов, 126 символов
2. test_files/file2.txt: 17 слов, 87 символов
3. test_files/file3.txt: 19 слов, 198 символов

Итог: 57 слов, 411 символов.
```

Если файл не удалось прочитать, вместо цифр выводится сообщение об ошибке, напр.: `3. some.txt: Ошибка - Cannot open file: some.txt`.
